# Gestion des erreurs

Une erreur (ou exception) est un évènement qui interromp le déroulement normal d'un programme.

# Pourquoi gérer les erreurs

1. Eviter le crash du programme
2. Améliorer l'expérience utilisateur
3. Contrôler le flux d'exécution
4. Anticiper les cas limites

# Types d'erreurs courants

- ValueError. Ex: `int("abc")`
- ZeroDivisionError. Ex: 10/0
- TypeError. Ex: "5" + 5
- FileNotFoundError
- IndexError. Ex: Index hors limites liste[100]
- KeyError. Ex: dico["cle_inexistante"]
